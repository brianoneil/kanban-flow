# Project Based Release Notes

## Summary

A Running list of features and fixes for the project.  This is not a complete list of changes, just the major ones that impact the user experience.  For a complete list of changes, see the git commit log.  This is meant to be a capture of the items in a release.  It should have a separate display of a markdown formatted list of changes for a release and support both human and agent editing of the items through the UI and MCP interface to alight with the existing functionality of the project.

## Requirements
- Release notes should be specific to the project and not include any other projects or general information.
- The notes should be written in markdown and support basic markdown formatting.
- The notes should be able to be edited by both humans and agents.
- The notes should include references to any related stories linked in a format of [story-id](story-url).  
- The notes should be searchable in the UI and MCP interface.
- The notes should be able to be downloaded as text, markdown, pdf, or JSON from the UI or through the MCP interface.
- The notes should have their own column in the UI and be able to be made to resize to fill the screen.
